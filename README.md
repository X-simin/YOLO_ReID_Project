# Pedestrian Detection & Cross-Frame ReID Tracking System

## 项目简介

- 基于 YOLOv8 + OSNet 的行人多目标跟踪与重识别 Web 系统。
- 本项目完整实现「行人检测 - 特征提取 - 跨帧匹配跟踪 - 人流统计可视化」全链路工程化代码，基于公开行人重识别数据集 Market1501 完成 OSNet 模型微调训练，使用 Flask 搭建前后端交互网页，可本地部署上传图片 / 监控视频，自动输出带唯一稳定 ID 的标注画面、人流统计图表。
- 项目针对性解决传统跟踪方案三大工业痛点：**行人 ID 频繁跳变**、**画面残留幽灵残影框**、**统计人数虚高失真**，适配校园 / 园区监控真实场景，完整覆盖数据集训练、算法优化、工程部署、Bug 迭代全流程，适合计算机视觉、深度学习算法岗求职展示。

## 核心亮点 

1. **完整 ReID 训练链路**：独立完成 OSNet 网络在 Market1501 数据集的数据增强、联合损失训练、离线指标测试，模型 Rank-1=92%，mAP=80%；
2. **自研鲁棒跟踪匹配逻辑**：采用「特征相似度为主 + IOU 为辅」匈牙利匹配，滑动窗口平滑特征，解决遮挡 / 框抖动导致 ID 跳变；
3. **根源消除幽灵框残影**：增加绘制层校验，仅渲染当前帧检测匹配成功目标，彻底消除目标离开后的残留空框；
4. **多层检测过滤抑制误检**：YOLO 检测后增加置信度 / 人体宽高比 / 最小尺寸三重过滤，过滤阴影、杂物、极小噪点；
5. **端到端 Web 工程落地**：基于 Flask 实现文件上传、视频批量推理、结果存储、Chart.js 数据可视化，可一键本地部署；
6. **完整迭代排错记录**：文档记录开发中所有核心 Bug、复现现象、定位过程与最终修复方案，体现问题排查能力。

## 技术栈 

### 1. 深度学习算法层

- **目标检测**：YOLOv8n (ultralytics) 轻量化实时行人检测
- **行人重识别**：OSNet (Omni-Scale Network) 多尺度轻量化 ReID 网络
- **匹配跟踪**：匈牙利线性分配算法、余弦相似度特征匹配、滑动窗口特征均值平滑
- **训练损失**：交叉熵分类损失 + 三元组对比损失联合优化
- **框架**：PyTorch 1.13+、CUDA GPU 加速推理

### 2. 图像处理 & 后端工程

- **图像处理**：OpenCV-Python、PIL、Numpy
- **Web 服务**：Flask 轻量后端接口
- **前端可视化**：原生 HTML、Chart.js（柱状图 / 饼图人流统计）

### 3. 数据集

- **Market1501**：标准行人重识别公开数据集（1501 个独立行人，32668 张多角度监控图像）

## 数据集 & 模型训练说明 (Market1501 Train)

### 1. 数据集介绍

Market1501 是行人重识别领域经典基准数据集：

- **样本规模**：1501 个独立行人，6 个不同监控摄像头拍摄，总计 32668 张标注图
- **场景特性**：天然包含视角偏移、光照变化、行人遮挡、侧身 / 弯腰多姿态，贴近真实校园监控场景
- **标准划分**：Query 查询集 (3368 张检索图) / Gallery 图库 (15913 张候选匹配库)

### 2. 完整训练流程

- **数据预处理增强**：随机水平翻转、尺寸归一化、中心裁剪，抑制过拟合，提升泛化能力；
- **模型微调策略**：基于 OSNet 开源预训练权重，在 Market1501 完整数据集微调 20 轮；
- **联合损失优化**：分类交叉熵损失 + 三元组损失联合训练，同时保证分类精度与特征区分度；
- **离线测试指标**：
  - **Rank-1 准确率**：92%（92% 检索样本首位匹配到同一行人）
  - **mAP 平均精度**：80%（全局检索排序综合精度，衡量模型整体区分能力）

### 3. OSNet 网络核心优势

采用多分支 OSBlock 轻量化残差块，4 条分支构建多尺度感受野，搭配共享 AG 自适应聚合门融合全局轮廓 + 局部衣物纹理特征，参数量仅 2.2M，兼顾精度与推理速度，适合视频实时跟踪场景。

## 核心算法流程 (Algorithm Pipeline)

### Step 1: YOLOv8 行人检测 + 多层无效框过滤

原始 YOLO 输出大量背景误检框，设计三层过滤逻辑：

1. **置信度过滤**：剔除低置信度阴影、杂物、远处噪点；
2. **人体宽高比过滤**：根据人体竖长特征，过滤扁形非人形框；
3. **边界 + 最小尺寸过滤**：剔除画面越界、尺寸过小的无效检测框。

### Step 2: OSNet 行人特征提取

裁剪检测框内行人图像送入 OSNet，输出 256 维归一化外观特征，用于跨帧身份匹配。

### Step 3: 跨帧多目标跟踪匹配（核心自研逻辑）

- **特征滑动窗口平滑**：保存连续 5 帧行人特征取均值，消除单帧检测抖动带来的特征突变；
- **混合代价矩阵匹配**：余弦相似度权重 0.8，IOU 位置权重 0.2，优先依靠行人外观匹配，弱化框小幅偏移影响；
- **跟踪器生命周期管理**：目标连续未匹配最大存活 15 帧，超过自动删除，兼顾短时遮挡 ID 稳定与长期消失无残影；
- **绘制层强约束（根治幽灵框）**：仅绘制当前帧检测成功匹配的跟踪目标，已离开画面、未匹配的目标不渲染，彻底消除空地残影框。

### Step 4: 自动人流统计与可视化

逐帧统计实时行人数量，视频结束后自动计算：

- 总独立行人数、单帧最大人数、平均每帧人数；
- 生成逐帧人数变化柱状图、有人 / 空帧占比饼图，前端可视化展示。

## 开发过程核心 Bug & 解决方案

### Bug 1：视频后半段大量幽灵残影框（空地漂浮空框）

- **现象**：行人走出画面后，对应 ID 框持续残留多帧，画面出现大量无行人的空框；
- **根因**：跟踪器仅做列表删除限制，未限制绘制逻辑，丢失未删除的目标仍会渲染；
- **修复方案**：新增绘制层校验，仅渲染 `miss_frames=0` 当前帧匹配成功的目标，丢失帧不再绘制，从可视化层面彻底消除残影。

### Bug 2：同一行人频繁跳变 ID，总独立行人数虚高

- **现象**：行人短暂遮挡、检测框短暂消失后，重新出现直接分配全新 ID，统计人数严重失真（如实际 9 人统计出 34 人）；
- **根因 1**：目标最大丢失帧 `MAX_MISS_FRAMES` 设置过小，遮挡后直接删除跟踪器；
- **根因 2**：匹配代价 IOU 权重过高，框轻微偏移直接匹配失败；
- **修复方案**：延长目标存活至 15 帧，调整匹配权重为特征 0.8 + IOU 0.2，增加 5 帧特征滑动窗口平滑，遮挡后可复用原有 ID。

### Bug 3：`IndexError: list index out of range` 索引越界崩溃

- **现象**：视频处理中途程序直接报错退出，跟踪列表索引访问越界；
- **根因**：匹配成功的跟踪器索引集合，在跟踪列表过滤删除后，旧索引失效；
- **修复方案**：增加索引合法性判断，遍历绘制前校验索引不超出列表长度，规避列表动态删减带来的下标失效问题。

### Bug 4：YOLO 误检地面杂物、垃圾桶、树木阴影为人

- **现象**：画面背景杂物持续生成检测框，增加 ReID 计算量，干扰跟踪匹配；
- **修复方案**：新增三层过滤机制，置信度阈值、人体宽高比、最小像素尺寸联合过滤，过滤 90% 以上背景误检。

### Bug 5：Market1501 训练后模型 Rank-1 精度偏低

- **现象**：外观相近的行人容易跨 ID 匹配，跟踪混淆；
- **根因**：仅使用单分类损失训练，缺少对比损失，特征区分度不足；
- **修复方案**：改用交叉熵 + 三元组损失联合训练，增大正负样本特征距离，提升特征辨识度。
